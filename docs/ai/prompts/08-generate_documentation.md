# Generate Documentation

Generate or update project documentation.

Before writing documentation:

Review:

- Source code
- Project memory
- Existing documentation
- Architecture

Documentation must:

- Reflect the current implementation.
- Avoid speculation.
- Be concise.
- Be organized.
- Be written for the intended audience.

Update only the documentation affected by the requested changes.

If implementation and documentation disagree, identify the discrepancy instead of guessing.

Record important documentation updates in project memory when appropriate.
